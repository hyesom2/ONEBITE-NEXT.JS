### ※ SSR(Server Side Rendering, 서버 사이드 렌더링) 소개 및 실습
- 가장 기본적인 사전 렌더링 방식
- 요청이 들어올 때 마다 사전 렌더링을 진행

```tsx
export const getServerSideProps = () => {}
```
- `getServerSideProps`
  - Next.js는 페이지 역할을 하는 파일 안에 `getServerSideProps`같은 약속된 이름의 함수를 만들어서 export하면 해당 페이지는 이제부터 SSR로 동작하도록 자동으로 설정
  - 실행 순서
  1. index 페이지 경로로 요청이 들어온다.
  2. getServerSideProps 함수가 동작을 해서 백엔드 서버로부터 데이터를 불러온다.
  3. 페이지 컴포넌트가 실행된다.
  - 예&#41; localhost:3000번의 index 경로로 index페이지를 요청해서 Next서버가 사전 렌더링을 하게 될 때 페이지 컴포넌트보다 먼저 실행이 되어서 인덱스 페이지 컴포넌트에 필요한 데이터를 또 다른 백엔드 서버로부터 불러오는 기능을 하는 함수

```tsx
// 예) hello 값을 서버로부터 불러왔다고 가정
export const getServerSideProps = () => {
  console.log('서버사이드프롭스에요');

  const data = 'hello';

  return {
    props: {
      data,
    }
  };
}

export default function Home({ data }) {
  console.log(data);

  return (
    ...
  )
}
```
- getServerSideProps 함수 안에서 return 객체 안에 props 프로퍼티로 객체로 넣어준다.
- props 프로퍼티로 데이터 객체를 넣어줘서 데이터를 컴포넌트 페이지에게 전달한다.

