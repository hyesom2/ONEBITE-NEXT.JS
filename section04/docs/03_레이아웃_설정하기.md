# 3. 레이아웃 설정하기

### ① /search 의 경우
```
app
 ┣ page.tsx ----------→ /
 ┣ search
 ┃ ┣ page.tsx --------→ /search
 ┃ ┗ layout.tsx ------→ /search 페이지의 레이아웃으로 자동 적용 
```

- /search 페이지는 레이아웃 컴포넌트가 먼저 렌더링이 되고 그 안에 `children`으로 페이지 컴포넌트가 렌더링됨

### ② /search/setting 의 경우
```
app
 ┣ page.tsx ----------→ /

 ┃ ┣ page.tsx --------→ /search
 ┃ ┣ layout.tsx ------→ /search 페이지의 레이아웃으로 자동 적용 
 ┃ ┣ setting
 ┃ ┃ ┗ page.tsx ------→ /search/setting
```

<img width="1194" height="510" alt="image" src="https://github.com/user-attachments/assets/fb4f91e4-f1c9-47e7-820b-1ba2017cb049" />

- `/search`로 시작하는 경로이므로 레이아웃 자동 적용 
- /search/setting 페이지는 레이아웃 컴포넌트가 먼저 렌더링 되고 레이아웃 안에 `children`으로써 setting 페이지 컴포넌트가 렌더링됨
- `/search` 경로로 시작하는 모든 페이지의 레이아웃으로 적용됨 → `/search/*`

---

### ③ /search/setting 안에 layout이 추가될 경우
- 레이아웃이 자동으로 중첩되어서 적용됨

```
app
 ┣ page.tsx ----------→ /
 ┣ search
 ┃ ┣ page.tsx --------→ /search
 ┃ ┣ layout.tsx ------→ /search 페이지의 레이아웃으로 자동 적용 
 ┃ ┣ setting
 ┃ ┃ ┣ page.tsx ------→ /search/setting
 ┃ ┃ ┗ layout.tsx
```

<img width="1201" height="595" alt="image" src="https://github.com/user-attachments/assets/ee250753-4dfc-43ae-8391-93c0cb95a1b2" />

- /search 폴더 아래의 layout이 먼저 적용이 되고, /search/setting 폴더 아래의 layout이 중첩이 되어서 적용이 된 다음에 마지막으로 setting 페이지 컴포넌트가 렌더링됨

---

### ※ 레이아웃
```tsx
// /search/layout.tsx

export default function Layout() {
  return (
    <div>
      <div>임시 서치바</div>
    </div>
  )
}
```
- `/search`로 시작하는 모든 경로의 페이지에 공통적으로 나타난다.
- 하지만 이렇게 하면 레이아웃은 렌더링이 되지만 페이지가 렌더링 되지 않는다. <br />
  → 레이아웃 컴포넌트 안에 페이지 컴포넌트는 어디에다가 렌더링을 할 것인지 설정을 안했기 때문

```tsx
// /search/layout.tsx

export default function Layout({ children }: { children: React.ReactNode}) {
  return (
    <div>
      <div>임시 서치바</div>
      {children}
    </div>
  )
}
```
- layout 컴포넌트에서는 자동으로 `children`이라는 이름의 props로 현재 페이지 컴포넌트가 전달이 되기 때문 <br />
  → layout 컴포넌트의 return문 안에 페이지 컴포넌트가 어디에 배치할 것인지 직접 설정

### ※ 라우트 그룹(Route Group)
- 예&#41; book 페이지를 제외한 index 페이지와 search 페이지에만 공통적으로 적용되는 레이아웃 설정
- 경로와 관게없이 그냥 특정 페이지들에만 적용이 되는 `공통 레이아웃` 
- app폴더 밑에 새로운 폴더를 생성 <br />
  → 새로운 폴더의 이름을 `소괄호()`로 감싼다.

```
app
 ┣ (with-searchbar)
 ┃ ┣ search
 ┃ ┃ ┗ page.tsx
 ┃ ┣ layout.tsx
 ┃ ┣ page.module.css
 ┃ ┗ page.tsx
 ┣ book
 ┃ ┗ [id]
 ┃ ┃ ┗ page.tsx
```
- `라우트 그룹(Route Group)`폴더는 경로상에는 아무런 영향을 주지 않음
- 라우트 그룹(Route Group)폴더 안에 새로운 레이아웃 파일을 만들면 해당 레이아웃 파일은 index 페이지와 search 페이지에 공통적으로 다 적용
