# 2. 페이지 라우팅 설정하기

### ※ Page Router 버전의 페이지 라우팅
- 생성해둔 파일들의 이름이 곧 경로가 된다.

```
pages
 ┣ index.tsx -----→ /
 ┗ search.tsx ----→ /search
```

### ※ App Router 버전의 페이지 라우팅
```
app
 ┣ page.tsx ----------→ /
 ┣ search
 ┃ ┗ page.tsx --------→ /search
 ┣ book
 ┃ ┣ page.tsx --------→ /book
 ┃ ┣ [id]
 ┃ ┃ ┗ page.tsx ------→ (동적경로)/book/1, 2, 3 ... 100
```

---

### ※ 쿼리 스트링(Query String)
- 경로 끝에 `?q=한입` 처럼 물음표와 함께 명시되기 때문에 라우팅에는 전혀 영향을 주지 않음
- App Router에서는 `쿼리스트링`이나 `URL 파라미터`와 같은 경로상에 포함되는 값들은 페이지 컴포넌트에게 모두 `props`로 전달
```tsx
// /search/page.tsx

export default function Page(props) {
  console.log(props);

  return (
    <div>search 페이지</div>
  )
}
```

```tsx
export default async function Page({ searchParams }: { searchParams: Promise<{ q: string }>}) {
  const { q } = await searchParams;
  
  return (
    <div>Search 페이지 : {q} </div>
  )
}
```
- searchParams 객체로부터 실제 쿼리 스트링 값을 꺼내서 활용하려면 타입 정의를 해야함<br />
  → props로 searchParams 객체가 들어오고`({ searchParams })` <br />
  → searchParams의 타입은 Promise객체 형태로 그 안에 q는 string이라는 결과 값이 포함`({ searchParams : Promise<{ q: string }>})`
- 페이지 컴포넌트 안에서 `const { q } = await searchParams;`로 q의 값을 꺼내온다. <br />
  → 구조분해할당 <br />
  → `await 키워드` : 페이지 컴포넌트가 이루고 있는 함수가 `async`키워드를 붙여 활용

```
❓ 어떻게 함수형 컴포넌트에 async 키워드를 붙여서 사용할 수 있나요?
→ 리액트의 서버 컴포넌트 이기 때문
→ 서버 컴포넌트의 경우 서버 측에서 사전 렌더링을 위해서 한 번만 실행되기 때문에 비동기적으로 실행이 되어도 문제 발생X
```

### ※ 동적 경로(Dynamic Routes)
```
app
 ┣ book
 ┃ ┣ [id]
 ┃ ┃ ┗ page.tsx ------→ (동적경로)/book/1, 2, 3 ... 100
```

```tsx
export default function Page() {
  return (
    <div>book/[id] 페이지</div>
  )
}
```
- localhost:3000/book/1, /book/2, ... /book/100, /book/200 어떤 값이든 위의 페이지에 대응
- URL 파라미터를 포함하고있는 `동적 경로 라우팅`을 설정할 때 App Router 버전에서는 `대괄호`와 함께 URL 파라미터를 명시하면 쉽게 설정됨

---

### ※ URL 파라미터

```tsx
export default async function Page({ params }: { params: Promise<{ id: string }> }) {
  const { id } = await params;

  return (
    <div>book/{id} 페이지</div>
  )
}
```

- URL 파라미터를 꺼내올 땐 props로 부터 `params`값을 꺼내서 활용하려면 타입 정의를 해야함<br />
  → props로 params 객체가 들어오고`({ params })` <br />
  → params의 타입은 Promise객체 형태로 그 안에 id는 string이라는 결과 값이 포함`({ params: Promise<{ id: string }>})`
- 
- 페이지 컴포넌트 안에서 `const { id } = await params;`로 id의 값을 꺼내온다. <br />
  → 구조분해할당 <br />
  → `await 키워드` : 페이지 컴포넌트가 이루고 있는 함수가 `async`키워드를 붙여 활용
- `[id]` : 딱 하나의 URL 파라미터에만 대응하도록 설정 <br />
  → 예) /book/1, /book/2, ... /book/100
- `[...id]` : URL 파라미터가 중첩으로 여러개 전달되는 경로에도 대응 <br />
  → 캐치 올 세그먼트(Catch All Segment) <br />
  → 예) /book/1/2, /book/1/2/3, ... /book/1/2/3/4/5

```
⚠️ 주의!
- 캐치 올 세그먼트(Catch All Segment)를 사용하여 경로를 설정할 경우 URL 파라미터가 아예 존재하지 않는 값에 대응이 되지 않음
→ 옵셔널 캐치 올 세그먼트(Optional Catch All Segment)로 만든다.
→ [[...id]]
```
