# 1. 스트리밍(Streaming)이란?

<!-- 이미지1 -->
① 서버에서 클라이언트로 데이터를 넘길 때 데이터의 크기가 크거나 서버 측에서 데이터를 준비하는데 걸리는 시간이 오래걸려서 데이터를 빠르게 전송하기 어려움

<!-- 이미지2 -->
② 데이터를 `여러 개의 조각으로 쪼갠 후` 쪼개진 작은 용량의 데이터들을 하나씩 클라이언트에게 전송하는 기술

<!-- 이미지3 -->
③ 클라이언트의 입장
- 모든 데이터가 다 불러와지지 않은 상태에서도 지금까지 전달받은 데이터에 접근할 수 있음
- 결과적으로 사용자에게 긴 로딩없이 좋은 경험을 제공할 수 있음

<!-- 이미지4 -->
④ 자체적으로 스트리밍 기능을 제공함 (출처 : nextjs.org) : 동영상뿐만이 아닌 일반적인 웹 서비스에서도 동일하게 HTML 페이지를 스트리밍 가능

<!-- 이미지5 -->
⑤ 사용자가 특정 페이지에 접속했을 때 일단 빠르게 렌더링이 오래걸리지 않는 단순한 컴포넌트들부터 먼저 화면에 보여줌

<!-- 이미지6 -->
⑥ 데이터 페칭 등의 이유로 렌더링이 오래 걸릴 것으로 예상되는 컴포넌트들은 로딩바 같은 대체 UI를 화면에 보여줌

<!-- 이미지7 -->
⑦ 서버 측에서 컴포넌트의 렌더링이 완료되면 컴포넌트들만 따로 전달하여 컨텐츠를 보여줌으로써 페이지를 제공

### 예&#41; Search 페이지의 경우

<!-- 이미지8 -->
- 루트 레이아웃 컴포넌트 : 전체 페이지에 레이아웃을 설정
- 서치바 레이아웃 컴포넌트 : 서치바 컴포넌트를 렌더링
- 서치바 컴포넌트 : 실제 서치바 컴포넌트
- 페이지 컴포넌트 : 백엔드 서버로부터 비동기적으로 검색 결과 데이터를 불러옴

### ※ 페이지 스트리밍
- 오래 걸리는 컴포넌트의 렌더링을 사용자가 좀 더 좋은 환경에서 기다릴 수 있도록
- 빨리 렌더링 할 수 있는 컴포넌트들을 밑반찬 처럼 내주는 것
