# 3. 스트리밍(3) : 컴포넌트 스트리밍 적용하기
- React의 `Suspense 컴포넌트`를 이용하여 컴포넌트 단위로 더 세밀하게 스트리밍

### ※ 페이지 스트리밍의 한계

```
search
 ┣ loading.tsx
 ┗ page.tsx
```
```tsx
export default function Loading() {
  return <div>Loading ...</div>
}
```
- 페이지 컴포넌트에 스트리밍을 적용해주기 위해 페이지 컴포넌트와 동일한 경로에 `loading.tsx`파일 생성
- loading.tsx파일 생성 후, 대체 UI 역할 컴포넌트를 작성
- 하지만 `loading.tsx`파일을 이용하는 방식은 무조건 페이지 컴포넌트 단위로만 스트리밍을 적용 <br />
  → 예&#41; book-item, searchbar 등의 일반 컴포넌트들에 따로 스트리밍을 할 수 없음 <br />
  → 세밀하게 스트리밍 설정 불가능

---

### ※ 컴포넌트 스트리밍
- 예&#41; search 페이지에 적용

```tsx
import BookItem from '@/components/book-item';
import { BookData } from '@/types';
import { delay } from '@/util/delay';
import { Suspense } from 'react'; // 4.

// 1.
async function SearchResult({ q }: { q: string }) {
  await delay(2000);
  const response = await fetch(`${process.env.NEXT_PUBLIC_API_SERVER_URL}/book/search?q=${q}`, { cache: 'force-cache'}); // 2.
  if (!response.ok) {
    return <div>검색 중 오류가 발생했습니다...</div>
  }
  const books: BookData[] = await response.json();
  
  return (
    <div>
      {
        books.map((book) => (
          <BookItem key={book.id} {...book} />
        ))
      }
    </div>
  )
}

export default async function Page({ searchParams }: { searchParams: Promise<{ q?: string }> }) {
  const { q } = await searchParams;

  return (
    // 5~6.
    <Suspense key={q || ''} fallback={<div>Loading...</div>}>
      <SearchResult q={q || ''} /> // 3.
    </Suspense>
  )
}
```
1. 페이지 컴포넌트 내부의 비동기 작업은 하는 페이지 컴포넌트의 하위 컴포넌트로 분리
2. SearchResult 컴포넌트는 searchParams.q를 
3. 페이지 컴포넌트에서 return에 SearchResult를 반환한다.
4. React로 부터 Suspense라는 컴포넌트를 불러온다.
5. Suspense 컴포넌트로 비동기 작업을 하는 SearchResult 컴포넌트를 감싼다. <br />
  → SearchResult 컴포넌트는 스트리밍이 자동으로 설정
6. `fallback` Props : 로딩 상태를 표시하는 대체 UI, 컴포넌트를 전달해도 똑같이 동작

### ※ 쿼리스트링만 변경이 되었을 경우 문제점 해결 방법
- 페이지 스트리밍에서 사용되는 `loading.tsx`파일과 동일하게 브라우저가 `쿼리스트링만 변경`이 되었을 경우, 로딩 상태가 표시 되지 않는 문제점이 발생한다.
- `Suspense 컴포넌트에 key Props 전달` <br />
  → key 값이 변화할 때 마다 다시 로딩 상태로 돌아가도록 설정 <br />
  → query string의 `q`값이 변할 때 마다 로딩 상태를 다시 표시 <br />
  → `key={q || ''}`
