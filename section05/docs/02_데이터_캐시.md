# 2. 데이터 캐시(Data Cache)
- fetch 메서드를 활용해 불러온 데이터를 Next 서버에서 보관하는 기능
- 영구적으로 데이터를 보관하거나, 특정 시간을 주기로 갱신 시키는 것이 가능
- 불필요한 데이터 요청의 수를 줄여서 웹 서비스의 성능을 크게 개선할 수 있음

```tsx
const response = await fetch(`~/api`); // 기존에 사용하던 fetch 메서드

const response = await fetch(`~/api`, { cache: "force-cache" }); // 두 번째 인수에 객체 형태의 추가적인 옵션 설정
```

- 두 번째 인수로 `객체 형태`의 추가적인 옵션을 설정
- 데이터 캐시의 옵션들은 오직 fetch 메서드에서만 활용 가능 → Axios 등의 다른 HTTP 리퀘스트 라이브러리에서는 활용X
- 일반적인 fetch 메서드가 아닌 Next.js가 자체적으로 데이터 캐시와 관련된 여러 개의 기능을 덧붙여 놓은 확장판 개념의 fetch 메서드 

### ① { cache: "force-cache" }

<img width="891" height="367" alt="image" src="https://github.com/user-attachments/assets/0e5acb89-4e4e-4dd3-9211-88c8ced0fad9" />

```tsx
const response = await fetch(`~/api`, { cache: "force-cache" });
```
- 요청의 결과를 무조건 캐싱함
- 한번 호출 된 이후에는 다시는 호출되지 않음

### ② { cache: "no-store" }

<img width="891" height="339" alt="image" src="https://github.com/user-attachments/assets/4d9f51a2-f496-4b56-a1da-0d666ef5ef77" />

```tsx
const response = await fetch(`~/api`, { cache: "no-store" });
```
- 데이터 페칭의 결과를 저장하지 않는 옵션
- 캐싱을 아예 하지 않도록 설정하는 옵션
- Next.js 15버전부터는 `데이터 캐시의 기본값`
